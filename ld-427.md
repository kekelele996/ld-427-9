## 项目主要需求

### 项目概述

【纯后端服务】装修预算与支出管理 API 服务。面向装修公司和业主提供预算编制、支出审批、供应商管理和对账结算的 RESTful API。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿后端全链路）

1. **预算表（BudgetSheet）** — 字段：所属项目ID、预算名称、总金额、已支出金额、冻结金额（审批中的支出）、可用余额（自动计算）、状态（Draft/Active/Locked/Archived）、创建人ID、审批人ID、版本号
   - 关联：1个预算表 → 多个预算项
   - **贯穿后端**

2. **预算项（BudgetItem）** — 字段：所属预算表ID、类别（Design/Material/Labor/Furniture/Appliance/Contingency/Other）、子类名称、预算金额、已支出金额、差异金额、排序、备注
   - **贯穿后端**

3. **支出记录（ExpenseRecord）** — 字段：所属预算项ID、支出金额、支出日期、支出方式（Cash/BankTransfer/Credit/Company）、供应商ID、发票号、支出描述、附件URL、审批状态（Draft/Submitted/Approved/Rejected/Paid）、申请人ID、审批人ID、审批意见、付款日期
   - **贯穿后端**

4. **供应商（Supplier）** — 字段：供应商名称、类别（Material/Furniture/Appliance/Labor/Design/Other）、联系人、联系电话、地址、开户行、银行账号、合作状态（Active/Suspended/Blacklisted）、评分（1-5）
   - **贯穿后端**

5. **对账单（Reconciliation）** — 字段：所属项目ID、对账期间（月份）、供应商ID、应付金额、已付金额、未付金额、对账状态（Pending/Confirmed/Disputed/Resolved）、确认人ID
   - **贯穿后端**

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / FinanceManager / ProjectManager / Accountant / Owner
   - 触达文件层：DB role表 + RBAC中间件 + controller权限检查 + 类型定义
   - 特殊规则：Owner只能查看不能操作，Accountant可以录入支出但不能审批

2. **审计日志**
   - 记录预算调整、支出审批、付款确认、对账操作
   - 触达文件层：DB audit_log表 + 日志中间件 + service埋点 + 类型定义

3. **接口限流**
   - 触达文件层：限流中间件 + Redis

### 共享枚举/常量（≥ 2 组）

1. **预算状态 BudgetStatus**：Draft / Active / Locked / Archived
   - 定义：`backend/src/types/enums.ts`

2. **支出审批状态 ExpenseStatus**：Draft / Submitted / Approved / Rejected / Paid

3. **支出方式 PaymentMethod**：Cash / BankTransfer / Credit / Company

4. **供应商状态 SupplierStatus**：Active / Suspended / Blacklisted

5. **预算类别 BudgetCategory**：Design / Material / Labor / Furniture / Appliance / Contingency / Other

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色权限校验
3. **`auditLogMiddleware`** — 审计日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`rateLimitMiddleware`** — 接口限流
6. **`requestLoggerMiddleware`** — 请求日志
7. **`validationMiddleware`** — 参数校验

### 技术要求

- **英文短名（用于 Docker Compose）**：`renovation-budget-api`
- **后端端口**：19306 / 3000
- **后端**：Django + Python（DRF）
- **数据库**：PostgreSQL 15
- **缓存**：Redis 7（限流 + 预算快照缓存）
- **认证**：JWT（djangorestframework-simplejwt）
- **API文档**：DRF Browsable API + Swagger（drf-spectacular）

### 部署要求

后端 + PostgreSQL + Redis，Docker Compose 编排。
- 顶层 `name: renovation-budget-api`
- 不写 `version:` 字段
- PostgreSQL + Redis 命名卷 + PostgreSQL healthcheck
- `.env` 必含：`COMPOSE_PROJECT_NAME`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `JWT_SECRET`, `REDIS_URL`, `BACKEND_PORT`

### Git 要求

`git init` + `.gitignore`。
